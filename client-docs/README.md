# 顧客資料回収システム

会計事務所向けの月次資料回収を効率化するための一式ツール。
ブラウザだけで動作（バックエンド不要、`localStorage` を使用）。

## 構成

| #  | 機能                           | ファイル                  |
|----|------------------------------|---------------------------|
| ①  | 顧客別 必要資料リスト              | `index.html` / `data.js`  |
| ②  | 提出フォーム（資料ごと）           | `submission-form.html`    |
| ③  | 毎月5日送信用 依頼メール 自動作成  | `email-generator.html`    |
| ④  | 提出状況の管理                    | `status-tracker.html`     |
| ⑤  | 不足・未提出の自動判定             | `status-tracker.html`     |
| ⑥  | 不足項目に紐づく督促文の自動作成   | `reminder.html`           |

## サンプル顧客

`data.js` の `INITIAL_CLIENTS` に **HRTコンサルティング株式会社** を1社登録済み。
必要資料は以下の3点（対象月: 6月）。

- (a) 預金のCSV
- (b) GMOあおぞらカード明細の6月使用分
- (c) 経費精算（6月使用分）の領収書

## 使い方

1. `index.html` をブラウザで開く（ローカルファイルでOK）。
2. 上部ナビゲーションから各機能へ。
3. データはブラウザの `localStorage` に保存される。

## 自動判定ロジック（⑤）

`data.js` の `detectMissing(client)` が以下をチェック:

- **未提出**: ファイルが1件もない
- **ファイル数不足**: `rules.minFiles` を下回る
- **拡張子不正**: `rules.extensions` 以外
- **対象月不一致**: ファイル名に `YYYYMM` / `YYYY-MM` / `M月` が含まれない（預金CSV・カード明細のみ）

## スケジュール送信（③）

cron 等で毎月5日にメール送信スクリプトを叩く想定。

```cron
0 9 5 * *  node send-monthly-request.js
```
